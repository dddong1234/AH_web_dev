# 6일차 폐렴예측 공통규칙

## 1. 이번 Stage 범위

이번 Stage 6의 목표는 저장된 진료기록의 X-ray 이미지를 활용해 AI 폐렴 예측 결과를 조회할 수 있도록 만드는 것이다.

- 진료기록 상세 기준으로 동작한다.
- 예측에는 진료기록 저장 시 업로드한 X-ray 이미지를 사용한다.
- 같은 모델로 저장된 예측 결과가 이미 있으면 재추론하지 않고 저장된 결과를 반환한다.
- AI 예측 결과는 목록으로 조회 가능해야 한다.

## 2. 공용 인터페이스 초안

공용 인터페이스는 모델추론 코드와 API 코드가 서로 맞물리도록 하는 공통 약속이다.

서비스 함수 초안:

```python
async def get_or_create_ai_analysis(
    db,
    record_id: int,
    model_name: str,
) -> dict:
    """
    1. record_id에 연결된 X-ray 이미지를 조회한다.
    2. 같은 record_id + model_name 조합의 저장 결과가 있으면 그 결과를 반환한다.
    3. 저장 결과가 없으면 AI 추론을 수행한다.
    4. 추론 결과를 저장한 뒤 반환한다.
    """
```

## 3. 공용 반환 형식

```json
    {
    "id": 1,
    "record_id": 10,
    "is_pneumonia": true,
    "confidence": 94.5,
    "heatmap_url": "/media/heatmaps/record-10.png",
    "ai_model": "team-model-v1",
    "created_at": "2026-07-22T10:00:00Z",
    "updated_at": "2026-07-22T10:00:00Z"
    }
```

- `id`: AI 분석 결과 ID
- `record_id`: 대상 진료기록 ID
- `is_pneumonia`: 폐렴 여부
- `confidence`: 예측 신뢰도
- `heatmap_url`: heatmap 이미지 URL, 없으면 `null`
- `ai_model`: 사용한 모델명
- `created_at`: 예측 수행 일시
- `updated_at`: 수정 시각

```json
  {
  "id": int,
  "record_id": int,
  "is_pneumonia": bool,
  "confidence": float,
  "heatmap_url": str,
  "ai_model": str,
  "created_at": str,
  "updated_at": str | None,
  }
```

## 4. 구현 시 공통 규칙

- 응답 필드명은 합의한 형식을 임의로 변경하지 않는다.
- 저장된 예측 결과 재사용 규칙을 반드시 지킨다.
- 요구사항 정의서에 없는 기능은 우선순위를 낮추고, 먼저 필수 요구사항부터 구현한다.
- 공용 인터페이스 변경이 필요하면 먼저 팀장과 공유한다.

## 5. merge 전 확인 항목

- 저장된 X-ray 이미지로 예측이 동작하는지
- 같은 모델 결과가 저장되어 있으면 재추론하지 않는지
- 응답 필드가 명세와 일치하는지
- API가 요구사항 정의서와 맞는지
